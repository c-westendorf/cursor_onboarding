# Self-Assessment Checklist

[Home](../README.md) > [Team Maturity](README.md) > Assessment Checklist

---

## How to Use

Check the items you can confidently answer "yes" to. Your level is the highest section where you've checked most items.

---

## Level 1: Beginner

### Basic Usage
- [ ] Can open projects and navigate in Cursor
- [ ] Use code completion regularly
- [ ] Know how to open chat (Cmd+L / Ctrl+L)
- [ ] Understand difference between ask and agent mode

### First Rules
- [ ] Have created at least one `.mdc` rule
- [ ] Understand basic frontmatter (description, alwaysApply)
- [ ] Can verify a rule is being applied

### Context Basics
- [ ] Use `@file` references in prompts
- [ ] Know what `.cursorignore` does
- [ ] Understand that rules consume context

**Level 1 Score:** ___ / 10

---

## Level 2: Intermediate

### Rule Types
- [ ] Know three rule types (always, auto-attach, on-demand)
- [ ] Use glob patterns for auto-attach rules
- [ ] Have rules organized by scope/domain
- [ ] Understand when to use each type

### Context Management
- [ ] Have optimized rules for token efficiency
- [ ] Use on-demand rules for detailed references
- [ ] Comprehensive `.cursorignore` configured
- [ ] Can audit which rules are active

### Decision Making
- [ ] Understand the knowledge codification decision flow
- [ ] Can decide if knowledge should be rule vs skill vs code
- [ ] Know when NOT to create a rule

**Level 2 Score:** ___ / 11

---

## Level 3: Advanced

### Workspace Organization
- [ ] Workspace organized by product, not function
- [ ] Directory structure matches project size
- [ ] Rules scoped appropriately with globs
- [ ] Documentation is @-referenceable

### MCP Integration
- [ ] Have at least one MCP server configured
- [ ] Understand MCP security considerations
- [ ] Know where to find MCP servers
- [ ] Can troubleshoot MCP connection issues

### Team Patterns
- [ ] Team has documented rule standards
- [ ] Rule review process exists
- [ ] Contributing to shared resources
- [ ] Can onboard new team members

### Advanced Workflows
- [ ] Use parallel agent approaches when appropriate
- [ ] Understand agent workflow modes
- [ ] Strategic about fresh dialogs vs continuing

**Level 3 Score:** ___ / 13

---

## Level 4: Expert

### Technical Leadership
- [ ] Have built custom MCP server
- [ ] Contributing rules to org marketplace
- [ ] Helping shape organizational standards
- [ ] Mentoring others on Cursor usage

### Innovation
- [ ] Identified novel use cases for Cursor
- [ ] Created reusable patterns/templates
- [ ] Contributed to Cursor community
- [ ] Evaluating new MCP servers/features

### Strategic Impact
- [ ] Cursor usage measurably improved team output
- [ ] Reduced repeated mistakes through rules
- [ ] Knowledge codification is part of team culture

**Level 4 Score:** ___ / 10

---

## Scoring Guide

| Level | Required Score |
|-------|---------------|
| Level 1 | 7+ / 10 |
| Level 2 | 8+ / 11 |
| Level 3 | 10+ / 13 |
| Level 4 | 7+ / 10 |

## Growth Planning

### If mostly Level 1:
**Focus areas:**
- Complete [First 30 Minutes](../01-getting-started/first-30-minutes.md)
- Create 2-3 basic rules
- Practice daily usage

**Timeline:** 1-2 weeks to Level 2

### If mostly Level 2:
**Focus areas:**
- Read [Context Budgeting](../03-rules-deep-dive/context-budgeting.md)
- Reorganize rules by type
- Learn decision flow

**Timeline:** 2-4 weeks to Level 3

### If mostly Level 3:
**Focus areas:**
- Set up MCP integrations
- Document team standards
- Start contributing to marketplace

**Timeline:** 1-2 months to Level 4

### If Level 4:
**Focus areas:**
- Mentor others
- Build custom servers
- Shape org standards
